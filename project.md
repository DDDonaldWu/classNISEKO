# classNISEKO – 專案說明文件

## 1. 專案目的（Purpose）
本專案為 **classNISEKO** 的官方形象網站，提供位於北海道二世谷（Niseko）的滑雪 / 單板滑雪課程資訊。

主要目標：
- 建立清楚、可信任的品牌形象
- 清楚說明課程內容與服務方式
- 蒐集潛在客戶的課程詢問
- 保持網站輕量、快速、好維護

本網站**刻意維持為靜態網站（Static Site）**，避免不必要的複雜度。

---

## 2. 目標使用者（Target Users）
- 第一次或初學滑雪 / 單板滑雪的學員
- 海外旅客（以中文、英文使用者為主）
- 主要以 **手機瀏覽** 的使用者

設計與功能優先順序：
- 清楚 > 花俏
- 安心感與信任感
- 行動裝置體驗

---

## 3. 技術架構與限制（Tech Stack & Constraints）

### 使用技術
- HTML / CSS / 原生 JavaScript（Vanilla JS）
- 無後端伺服器
- 使用 GitHub Pages 部署

### 明確限制（非常重要）
- ❌ 不使用前端框架（React / Vue / 等）
- ❌ 不引入建置流程（Build Step）
- ❌ 不使用後端 API
- ✅ 所有功能必須可由靜態檔案完成

所有程式碼需保持「**容易閱讀、容易修改**」。

---

## 4. 檔案結構與職責說明

```
/
├── index.html        # 主頁（預設語言）
├── index-en.html     # 英文版本
├── style.css         # 全站樣式（Mobile-first）
├── script.js         # 表單送出與互動邏輯
├── i18n.js           # 語言切換邏輯
├── sitemap.xml       # SEO
├── robots.txt        # SEO
├── CNAME             # 自訂網域
└── PROJECT.md        # 專案說明文件（本文件）
```

### 檔案使用原則
- `index.html` 與 `index-en.html` **共用同一組 CSS / JS**
- 避免在不同語言頁面中重複撰寫邏輯
- 所有樣式集中於 `style.css`
- 所有互動與資料處理集中於 `script.js`

---

## 5. 設計與 UX 原則

### 版面設計
- Mobile-first
- 內容最大寬度約 1200px
- 簡單、直覺的縱向閱讀流程

### 視覺風格
- 乾淨、冷靜、偏高級感
- 避免過多動畫與特效
- 版面配置優先使用 CSS，而非 JS

### 使用者體驗
- 明確的行動呼籲（CTA）
- 表單流程簡單、低摩擦
- 不加入不必要的步驟

---

## 6. 表單處理原則（Form Handling）

- 無後端系統
- 透過 JavaScript 將資料送出（如 Google Forms / Google Sheets）
- 必須支援：
  - Mobile Safari
  - Mobile Chrome

注意事項：
- 除非必要，不引入複雜的非同步流程
- 可接受「簡單但可靠」的失敗處理方式（alert / 訊息提示）

---

## 7. SEO 與可搜尋性

- `sitemap.xml` 手動維護
- `robots.txt` 允許搜尋引擎索引
- 優先使用語意化 HTML
- 網站速度為重要指標

## 9. 開發哲學（Development Philosophy）

本專案優先考量：
- **能快速完成並上線** > 完美設計
- **容易維護** > 技術炫技
- **實際可用** > 理論最佳解

如果一個解法：
- 3 分鐘內能看懂
- 修改時不會有心理壓力
- 在各裝置上穩定運作

那它就是目前階段的「好解法」。

---

## 10. 給未來協作者（包含 AI 🤖）的備註

- 除非明確需求，請勿引入框架或大型重構
- 避免為了「漂亮」而增加不必要的複雜度
- 所有建議以「小幅、可回滾」為優先
- 尊重既有結構，除非它真的阻礙擴充

---

## 最低維護規則（v1）

> 目的：避免過度重構、避免中英不同步

1. 中英文頁只負責結構，JS 不生成整個表單
2. 中英頁結構必須一致，文字可不同
3. JS 依賴的 id / class 兩頁都要存在
4. 不為了乾淨而重構
5. 每完成一個可獨立功能就停一次並 commit
